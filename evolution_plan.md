# 线虫OS演化模拟 — V2 升级计划

> 目标: 让AC/VU选主协议的更"逼真"，
> 通过分页迭代（跑→停→分析→继续）
> 在有限的2.9GB内存内模拟足够多的世代

## 当前模型的问题

1. **lin-12降到0.01成了最优解**
   - 因为模型里AC/VU是唯一的"测试用例"
   - 现实中lin-12还在很多其他决策用 (不止AC/VU)
   - 缺失约束: AC/VU只是lin-12众多作用中的一个

2. **两个细胞的N值在对称的终点稳定**
   - 正反馈不彻底——应该一个趋近1一个趋近0
   - 因为我的Hill方程参数设得太保守

3. **没有环境压力**
   - 只有"产生不对称=好"这一条
   - 现实中还有: 温度/食物/竞争者/随机噪声
   
## 要加的约束 (每个是一个可开关的模块)

### C1: 多任务约束 (lin-12在其他地方也需要)
- lin-12在多种细胞决策中都需要
- 优化lin-12的一个参数会影响所有任务
- 加两个"干扰测试": Notch在其他细胞的信号传递
- ❌ 最复杂的, 先不做

### C2: 能量的消耗约束
- 高表达配体/受体要消耗能量
- `lag2_expr_rate=3.2`太浪费了——3倍的配体表达
- 加入`energy_cost = sum(param - 1.0)`的平方惩罚
- 简单, 加一行fitness扣分

### C3: 随机噪声 (每次模拟用不同种子)
- 现在确定性太高——`np.random.seed(trial+42)`每次一样
- 改为: 每个个体用多个随机种子跑, 取平均fitness
- 测试鲁棒性——好的参数无论什么噪声都能正确决定

### C4: 多代跟踪 (演化轨迹保存)
- 每次run保存全种群参数快照到JSON
- 下次继续加载, 避免内存爆炸
- 引用计数释放

### C5: 基因之间互相依赖
- Notch信号强度不仅取决于配体, 还取决于细胞膜上的受体密度
- 受体内吞(temperature sensitive)和再循环
- 这个要重新写ODE

## 分页迭代计划

```
阶段1: 修复模型 [当前]
   加C2(能量惩罚) + C3(多噪声测试) → 跑200代(分4次, 每50代存一次)
   
阶段2: 灾难测试 [下一轮]
   加lin-12(lf)突变 → 看系统能否演化出补偿
   
阶段3: 基因复制 [再下一轮]
   模拟基因复制事件: 一个基因→两个副本→功能分化
   
阶段4: 环境切换 [最后]
   温度/食物/压力变化 → 看OS能否适应
```

## 存储策略

每50代 → `save_state_gen_XXX.json` (只存参数+适应性, ~10KB)
清理中间python对象

开始: 先做阶段1, 加能量惩罚+多噪声测试.
