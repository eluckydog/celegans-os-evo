## D1: 凋亡约束 — 分析

**修改**: 在 coupling_penalty 中, egl1_sens>3x AND cell_survive>3x 时加+1.0 penalty。

**效果**: ✅ 完全达到目的
- egl1_sens 从 19x → **2.9x** (受控)
- cell_survive 从 11.5x → **4.0x** (受控)
- 但 fitness 从 5.18 → **2.95** (大幅下降)

**trade-off 分析**:
- 凋亡约束强制系统放弃了Notch通路高表达策略
- lin12_expr → 0.1x, lag2_expr → 0.1x (Notch通路几乎被关闭)
- 系统转向: fork_rate↑, timer_speed↑, memory_stab↑ (替换策略)
- 只产生1个tip点 → 收敛极快，但卡在局部最优

**结论**: 约束有效但粗暴。直接抑制了 fitness 天花板。需要更精细的约束设计。

---

## D2: Notch自我抑制 — 分析

**修改**: 在 task_ac_vu 中加入 dN += -k*N*G (k=0.01, 高Notch抑制配体)。

**效果**: 
- best=4.507 (比baseline 5.18低13%)
- 但 **egl1_sens 和 cell_survive 仍然暴走到 9.8x+**
- tips=11（比baseline的9更多，说明搜索更活跃）
- lin12_expr 冲到 6.5x (比baseline 2.5x更高!)

**为什么自我抑制没控制住凋亡？**
- self-inhibition项太小(k=0.01)，在高Notch环境下被其他项压过
- 相反，自我抑制破坏了 AC/VU 分化的稳定性，迫使系统通过更大的 Notch 表达来补偿
- 更大的 Notch 表达 → 更大的细胞存活需求 → egl1_sens & cell_survive 暴走
- notch_thresh 下降 (-0.26) 但不够

**D2 证明了**: 单纯加微小的 Notch 负反馈，没有边界约束，反而恶化凋亡暴走。

---

## D3: 混合 — 分析

**修改**: D1的凋亡约束 + D2的自我抑制。

**效果**: 几乎与 D1 相同
- best=2.99 (vs D1的2.95)
- egl1_sens=1.48 (3.0x), cell_survive=3.22 (4.0x)
- 同样: lin12_expr / lag2_expr 降到 0.1x
- 转向 fork↑ / timer↑ / heritability↑

**D3 vs D1 差异点**:
- D3有 notch_sens = 2.04x (up), notch_decay = 1.22x (up)
- 提示：自我抑制在高 notch_sens 下还产生了一些正面影响
- tips=3 (vs D1的1, 更多探索)
- fitness 略高于 D1 (2.99 vs 2.95)

**结论**: 混合方案也没法绕过约束的 trade-off。凋亡约束是主导因素。

## 关键洞察

### 1. 凋亡约束的 root cause
baseline 中 egl1_sens=19x & cell_survive=11.5x 说明系统找到了一条"高压"策略:
- 高 Notch 信号 + 高 caspase 敏感性 → AC/VU 快速分化 → 高 fitness
- 代价: 细胞凋亡子系统推到极限
- D1加了约束后, 系统被迫放弃Notch通路, 用 fork+timer+memory 替代 → fitness 腰斩

### 2. Check noise子任务 (建议2验证)
在所有4个方向中:
- ach_sens: 稳定在 -0.46 ~ -0.53 (下降)
- gaba_sens: 稳定在 -0.50 ~ -0.54 (下降)
- **确认**: 系统确实在学习"忽略噪声"——这个趋势独立于任何修改, 是稳健的结果

### 3. Notch自我抑制 (建议3验证)
- k=0.01 太小, 不足以对抗高表达
- 需要 **k=0.05~0.1** 或 **非线性抑制** 才能产生 bistability

## 建议下一步

1. **D1再平衡**: 把约束阈值从 3x 放宽到 5x, penalty从1.0降到0.5。看看能否在不剧烈降低fitness的情况下控制极端值。
2. **D2增强版**: k_self_inhibit 从 0.01 提升到 0.1, 再加入滞后效应(延迟反馈)。真正创造bistability。
3. **新方向 D4**: 凋亡约束 + AC/VU 通路保护——当 Notch 通路被约束抑制时, 补偿 AC/VU 分化权重。
