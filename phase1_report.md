# 线虫OS 演化模拟器 — 阶段报告 v1

## 10方向总结

### ✅ 已确认 (不需要再跑)
| 方向 | 结论 |
|------|------|
| **1 — 基线 (Notch锁定)** | WT参数lin-12≈0.76, lag-2≈0.76, 趋近真实 |
| **2 — lin-12(lf)补偿** | ✅ 能补偿, 通过降notch_threshold |
| **3 — lag-2(lf)补偿** | ✅ 部分补偿, 比2更难 |
| **5 — 噪声鲁棒性** | ✅ 系统天然有韧性, 80%+正确率 |
| **6 — 自我不对称** | ✅ 纯噪声驱动的自发不对称 |

### ❌ 需要修正/排除的
| 方向 | 问题 |
|------|------|
| **4 — lin-12(gf)逃逸** | 恒为2个VU, 模型缺陷——缺少lin-12的非线性饱和 |
| **7 — 速度vs可靠性** | best=2.0满格——fitness函数漏洞: 阈值降到0.01就能瞬间决定 |
| **8 — 记忆持久性** | memory参数漂移到0.1——约束没锁住 |
| **9 — 双敲除** | 参数漂移逃脱fix——需要更严格约束 |
| **10 — evo-devo耦合** | 约束太弱, 没任何方向性 |

### 🔥 有趣的新方向 (值得再跑500代)

1. **双敲除的真正测试**: 把lin-12和lag-2完全锁在0.01, 放开所有其他参数, 看1,000代内是否真正能涌现补偿机制

2. **逆向演化**: 从lin-12(lf)起点开始→100代补偿→100代恢复到野生型→100代lin-12(gf)→追踪参数轨迹, 看演化是否需要回溯原路径

3. **二选一的tradeoff**: 一个子代表现为高lin-12(lf)模式, 另一个子代低lag-2模式, 两个都是"畸形"但互相补充。一起演化1000代, 看是否涌现分工

4. **三个细胞竞争**: 三个细胞争夺"谁变成AC"——当有多个竞争者时Notch侧向抑制如何演化

5. **环境切换**: 每30代切换环境(低噪声→高噪声→低噪声→高噪声), 看系统是否保留环境记忆, 还是每次都从头再来

6. **超参数敏感性分析**: 对同一个方向(比如方向2), 改变mutation_rate从0.05到0.3, 看最佳演化速度
