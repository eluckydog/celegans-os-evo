# DNA 作为编程语言 — 学习笔记 & 地图

## 核心隐喻

DNA ≈ 一个**4字符集、编译型、高度冗余、有反射机制**的编程语言。
运行这个"程序"的硬件是细胞（核糖体=CPU，RNA聚合酶=编译器）。

## 第一层：语法与编译

### 基础语法

| DNA概念 | 编程概念 | 举例 |
|---------|---------|------|
| 碱基 (A/T/C/G) | 字符集 (4个token) | 比任何编程语言的keyword都少 |
| 双链互补 | 严格的配对约束 | A↔T, C↔G (约束满足问题) |
| 5'→3'方向 | 代码从左到右读 | 反向互补链=逆序读 |
| 密码子 (Codon) | 指令/操作码 | 3核苷酸=1指令 (共64条) |
| 氨基酸 | 指令执行结果 | 20种氨基酸 (20种汇编指令) |
| 开放阅读框 (ORF) | 函数定义 | 起始ATG...终止TAA/TAG/TGA |
| 基因 | 函数/模块 | 约20,000-25,000个函数 |

### 编译流水线

```
DNA 源代码                    →  RNA 中间表示         →  蛋白质（可执行）
（双链，含内含子/外显子）       （单链，剪接后）       （折叠成3D结构）
  │                              │
  ▼ 转录                         ▼ 翻译
RNA聚合酶                       核糖体 (CPU核心)
编译器                          + tRNA (指令译码器)  
  │                              │
  ▼ 剪接                         ▼ 折叠
剪接体(去内含子→ 外显子拼接)     分子伴侣/自组装
                                  (链接器/运行时优化)
```

### 操作码表（64条指令 → 20种结果）

类似x86指令集，但某些"操作码"（Leucine, Serine, Arginine）有6个同义编码，类似语法糖/编译器优化选项。

```
AAA→Lys    AAG→Lys    (两个操作码, 同一结果)
TTT→Phe    TTC→Phe    (同理)
ATG→Met (唯一起始指令, 类似main函数入口)
TAA/TAG/TGA → STOP (3种return指令)
```

## 第二层：反射与元编程

### 表观遗传 ≈ 运行时反射

DNA的序列（源代码）不改，但**甲基化修饰**改变行为：
- 5mC (甲基化) → 基因沉默 / turn off
- 5hmC (羟甲基化) → 基因激活 / turn on
- DNMT酶 → 写入器 (Writer)、TET酶 → 擦除器 (Eraser)

这和Ub/SUMO竞争完全相同——就位点不同（CpG vs K位点），核心机制相同。

### 组蛋白修饰 ≈ 并发控制

组蛋白尾部可以有多种PTM（乙酰化/甲基化/磷酸化），决定DNA的"可读性"。
类似**数据库锁级别**——染色质开放=行级锁，紧密=表级锁。

## 第三层：版本控制与调试

### 变异 = 代码修改

| DNA | 编程 | 后果 |
|-----|------|------|
| SNP (单核苷酸多态) | 单字符修改 | 可能在编译阶段拦截（同义密码子） |
| Indel (插入/缺失) | 区块修改 | 通常导致frame shift → 完全的runtime崩溃 |
| CNV (拷贝数变异) | 函数重复/丢失 | 类似代码重复粘贴bug |
| 转座子 | 代码复制粘贴 | 移动代码段到新位置 |
| 基因重复 | fork | 一个副本保留，一个副本自由演化 |

### 冗余 = 编译器优化

遗传密码冗余度（64→20）的本质是：
1. **容错**：第3位置（摆动位置）变化通常不影响结果 → 类似编译器的-mtune优化
2. **调优**：高表达基因倾向使用特定"高频密码子" → 类似profile-guided optimization
3. **进化空间**：同义突变累积而不破坏功能 → 后续可以"解封"
